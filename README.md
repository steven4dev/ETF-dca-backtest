純靜態頁面，無後端，所有計算在瀏覽器即時完成。

## 計算邏輯

### DCA 模擬

每次扣款金額固定為 `currentAmt`，於目標日期（或下一個交易日）以當日收盤價買入：

```
持股數 += 每次金額 / 當日收盤價
```

### 報酬率

```
報酬率 = (最終市值 - 累計投入成本) / 累計投入成本 × 100%
```

### 年化報酬（CAGR）

```
CAGR = (最終市值 / 累計投入成本)^(1 / 實際年數) - 1
```

### 最大回撤（基於累積報酬率）

避免 DCA 持續注資掩蓋真實跌幅，改用報酬率曲線計算：

```
ret_i = (market_value_i - cost_i) / cost_i × 100

peak_ret = max(ret_0 ... ret_i)

drawdown_i = (ret_i - peak_ret) / (1 + peak_ret / 100)

max_drawdown = min(drawdown_0 ... drawdown_n)
```

### 佈局配置表加權

依各標的回測最佳報酬率正比分配，設 floor = 1 防止負報酬標的權重歸零：

```
weight_i = max(ret_i, 1) / Σ max(ret_j, 1)
```

## 分割自動補正原理

Yahoo Finance 的 Adj Close 只調整配息，不調整股票分割，分割日會有價格跳空。  
`auto_fix_splits()` 自動掃描並補正：

1. 找出單日跌幅 > 40% 的日期（正常市場崩跌通常不超過 20%）
2. 計算 `N = round(1 / ratio)`，確認誤差 < 12%
3. 將分割日**前**所有歷史價格除以 N
4. 循環掃描至無異常跳空為止（支援多次分割）

```
範例：
0052  分割前 248.50 → 分割後 35.57（−85.7%）
      補正後 35.50  →        35.57（+0.2%，正常波動）

00631L 分割前 469.35 → 分割後 20.13（−95.7%）
       補正後 20.41  →        20.13（−1.4%，正常波動）
```

## 新增 ETF

1. 在 `download_data.py` 的 `ETF_LIST` 加入新標的：
   ```python
   'XXXXX': 'XXXXX.TW',
   ```
2. 在 `generate_html.py` 的 `ETF_CONFIG` 加入：
   ```python
   'XXXXX': {'name':'XXXXX 顯示名稱', 'csv':'XXXXX_data.csv'},
   ```
3. 在 `ETF_COLORS`（JS 段落）加入對應顏色
4. 執行 `download_data.py` 與 `generate_html.py`
